# Systematic debugging

Reproduce the failure and identify the earliest incorrect state before editing.
Inspect relevant code paths, configuration, logs, and recent changes with narrow native tools.
Form one falsifiable hypothesis at a time and test it with fresh evidence.
Fix the systemic cause, preserve unrelated behavior, and verify both the reproduction and nearby regressions.
