# Repository exploration

Read project instructions first. Inspect Git status, the top-level tree, manifests, and focused symbols or searches.
Prefer targeted ranges and native search tools over recursive file dumps.
Trace the smallest relevant execution path and preserve provenance for conclusions.
Stop exploring once the task boundary and likely change surface are clear.
