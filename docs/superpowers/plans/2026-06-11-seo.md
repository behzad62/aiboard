# SEO Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add full technical SEO (metadata, OG/Twitter tags, JSON-LD, sitemap, robots, OG image) plus crawlable landing copy and an About page to the statically-exported client-side app.

**Architecture:** Each `"use client"` page is moved to a client component file and replaced by a thin server-component `page.tsx` that exports Next `Metadata`. All site identity comes from one new constants file (`lib/site.ts`) with a placeholder `SITE_URL`. The home server wrapper renders the hero `<h1>`, landing prose, and JSON-LD directly (server JSX → guaranteed in exported HTML). Sitemap/robots use Next's `app/sitemap.ts` / `app/robots.ts` conventions, which emit static files at build with `output: "export"`.

**Tech Stack:** Next.js 15 App Router (static export), React 19, TS strict, Tailwind. No new dependencies.

**Spec:** `docs/superpowers/specs/2026-06-11-seo-design.md`

**Important platform gotchas (from CLAUDE.md):**
- Never run `npm run build` while the dev server is running — it corrupts the dev server's `.next`. Stop the dev server first.
- There is no test runner; verification = `npm run build` + inspecting files in `out/` + `npm run lint`. TDD is replaced by build-output assertions for this plan (all changes are static markup/metadata).
- With default `trailingSlash: false`, the export produces `out/index.html`, `out/about.html`, `out/discussion.html`, `out/settings.html`, `out/sitemap.xml`, `out/robots.txt`.

export const dynamic = "force-static";

export default function sitemap(): MetadataRoute.Sitemap {

}

- [ ] **Step 2: Create `app/robots.ts`**

```ts
import type { MetadataRoute } from "next";
import { SITE_URL } from "@/lib/site";

// Required for output: "export" — generated once at build time.
export const dynamic = "force-static";

export default function robots(): MetadataRoute.Robots {
  // Allow everything: the tool pages are excluded via meta noindex, which
  // crawlers can only see if they are NOT robots-blocked.
  return {
    rules: { userAgent: "*", allow: "/" },
    sitemap: `${SITE_URL}/sitemap.xml`,
  };
}
```
